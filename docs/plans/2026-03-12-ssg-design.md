# Design Doc: Static Blog Generator (SSG) in Go

**Date**: 2026-03-12
**Status**: Approved
**Owner**: opencode

## Overview

This project implements a true Static Site Generator (SSG) in Go, with no external dependencies. It processes Markdown files from the `posts/` directory and generates a minimalist blog with an `index.html` at the root and post pages in the `static/` directory.

## Architecture

### Data Flow
1. **Source Discovery**: Scan `posts/*.md`.
2. **Parsing**: Extract YAML-style front-matter and parse Markdown to HTML.
3. **Data Aggregation**: Build a collection of post metadata and content, sorted by date.
4. **Rendering**: Interpolate data into embedded HTML templates using `html/template`.
5. **Output**: Write `index.html` to the root and each post to `static/[slug].html`.

### Project Structure
- `ssg/main.go`: The single-file entry point for the SSG.
- `posts/*.md`: Source Markdown content.
- `static/`: Destination for generated post pages and persistent assets (CSS, JS).
- `index.html`: The main landing page generated by the SSG.

## Design Components

### Core Logic (`ssg/main.go`)
- **`Post` Struct**:
  - `Title`, `Date`, `Description`, `Tags`, `Slug` (metadata).
  - `Content` (`template.HTML` for the body).
- **Markdown Parser**:
  - A custom `parseMarkdown` function using Go's `regexp` package to handle headings, lists, links, images, bold/italic text, and code blocks.
- **Front-Matter Parser**:
  - A simple function to split the `---` delimited YAML header from the content.
- **Embedded Templates**:
  - `layoutTmpl`: The base HTML layout including the `<head>`, `header`, `main`, and `footer`.
  - `indexTmpl`: The list of blog posts for the home page.
  - `postTmpl`: The template for individual post pages.
- **Build Process**:
  - `main()` orchestrates:
    - Creating/Ensuring the `static/` directory exists.
    - Loading and parsing all posts.
    - Rendering `index.html`.
    - Rendering individual post pages into the `static/` folder.

## Constraints & Considerations
- **No Dependencies**: Standard library only (`html/template`, `os`, `path/filepath`, `regexp`, `strings`, `time`).
- **Typography**: IBM Plex Mono must be used for the minimalist design.
- **Simplicity**: Favor regex-based parsing over a full Markdown spec implementation if it covers the project's needs.

## Success Criteria
- Running `go run ssg/main.go` generates a functional, responsive static blog.
- All links correctly point to `static/` for post pages and assets.
- `index.html` correctly displays a reverse-chronological list of posts.
